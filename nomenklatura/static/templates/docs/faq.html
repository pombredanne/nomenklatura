What is a dataset?

A dataset is a logical unit of entities in nomenklatura. Examples of this could be all politicians in a country, all countries in the world or all companies involved in a particular context.

Be aware that a dataset in nomenklatura doesn't have to correspond to a single data source. Names from a variety of origins can be combined and mapped together in a single dataset to integrate the data sources.

How do I add entities to a dataset?

The key function of nomenklatura is building entity lists dynamically. As a developer working with data, you would run a data extraction script and use the API to create these entities in nomenklatura. In a next step, you can review the newly created entities manually to remove duplicates and ambiguities. Finally, you would run your extraction script again, this time using the canonical entity names from nomenklatura to clean up your data.

That said, there are three methods for adding entities to nomenklatura:

  • Import bulk data from a CSV file. You can select to import data from a file, either to import new entities or alias lists. The process will ask you to define how each column in the imported file should be interpreted in the context of nomenklatura.
  • Use the API. Nomenklatura provides a full, REST-inspired API to provide data access. You can use this to find existing entities, but also the create new ones if no match can be found.
  • Add entities manually. On a dataset page, you can manually enter an entity name to create a new entity.

What is review mode?

In review mode, you are prompted to decide if an entity is an alias for another entity, whether it is invalid or whether it should remain as it is. To make this easier, each entity is compared to all others in the dataset, and the most similar entities are shown as choices for you to see if there is an obvious duplicate.

To make a change, just select the relevant option - another entity to merge with, or the invalid option, and save the result.

What does marking an entity as invalid mean?

Marking an entity as invalid is helpful when the entity was added to a dataset but doesn't actually make sense in the context of the dataset, i.e. when it is a category error.

For example, imagine importing speaker names from a parliamentary transcript into nomenklatura. Instead of being marked with a speaker name, some speeches may be marked in the transcript with a piece of pseudo-naming like Unknown, which doesn't actually represent a person's name.
